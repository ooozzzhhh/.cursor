# 网页信息存储规则

所有抓取结果存入 `info/` 目录，采用渐进式披露结构：索引 → 摘要 → 详情。

---

## 1. 目录结构

```
info/
├── 索引.md                    # 一级：总览，列出所有抓取任务
└── 来源_主题/                 # 按「来源+主题」分桶
    └── yyyyMMdd/              # 按抓取日期
        ├── 摘要.md             # 二级：本次抓取摘要
        └── 章节_xxx.md         # 三级：单章详情（按需创建）
```

### 1.1 命名规则

| 层级 | 命名 | 示例 |
|------|------|------|
| 来源_主题 | 来源简写_主题简写 | `SAP_生产计划MRP`、`MDN_JavaScript` |
| 日期目录 | yyyyMMdd | `20260305` |
| 摘要 | 摘要.md | 固定 |
| 章节 | 章节_序号或标题简写.md | `章节_01_净需求计算.md` |

---

## 2. 渐进式披露

| 层级 | 文件 | 内容 | 用途 |
|------|------|------|------|
| 1 | 索引.md | 任务列表：来源、主题、日期、摘要路径 | 快速浏览「抓过什么」 |
| 2 | 摘要.md | 本次抓取：URL、章节列表、核心要点、存储路径 | 了解「这次抓了什么」 |
| 3 | 章节_xxx.md | 单章完整正文 | 需要细节时再读 |

**原则**：先看索引，再看摘要，需要时再打开章节文件。

---

## 3. 文件内容模板

### 3.1 索引.md

```markdown
# 网页信息抓取索引

| 来源_主题 | 日期 | 摘要路径 |
|-----------|------|----------|
| SAP_生产计划MRP | 20260305 | [SAP_生产计划MRP/20260305/摘要.md](SAP_生产计划MRP/20260305/摘要.md) |
```

每次新抓取后，在索引中追加一行。

### 3.2 摘要.md

```markdown
# 抓取摘要

- **来源**：SAP Help Portal
- **URL**：https://help.sap.com/...
- **抓取时间**：2026-03-05
- **结束条件**：抓取 2 个章节

## 章节列表
1. 时间相关供应天数的使用示例
2. 净需求计算

## 核心要点
（简要归纳）

## 详情文件
- [章节_01_时间相关供应天数.md](章节_01_时间相关供应天数.md)
- [章节_02_净需求计算.md](章节_02_净需求计算.md)
```

### 3.3 章节_xxx.md

```markdown
# [章节标题]

**来源 URL**：...

## 正文
（从 snapshot 提取的完整内容）
```

---

## 4. 创建顺序

1. 若 `info/索引.md` 不存在，先创建
2. 创建 `info/来源_主题/yyyyMMdd/` 目录
3. 写入 `摘要.md`
4. 若章节内容较多或用户可能需细读，再创建 `章节_xxx.md`
5. 更新 `索引.md` 追加本次任务
